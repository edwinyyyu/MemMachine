# Mid-execution cues — substrate experiment (2026-04-28 continuation)

Continuation of `2026-04-24_mid_execution_cues_design.md`. Prior session ended at SA-full + multi-turn explicit thinking baseline of cov **0.876** / full_R@5 **0.646** on the 10 hard scenarios. This session tests two hypotheses: (1) native server-side reasoning instead of explicit-text thinking, and (2) "memory is cognition's substrate" — ingesting the agent's own THINKING/PROBE outputs back into EM as retrievable events.

All numbers gpt-5-mini, reasoning_effort=low, 10 hard scenarios (indices 10-19 of `data/mid_execution_scenarios.json`).

## Native reasoning vs explicit thinking

Mixed signal. Native is **+4.8pp** coverage on 9 overlapping scenarios but **−4.0pp** full_R@5 — coverage gain comes from cleaner step planning, R@5 loss comes from less-rich exec-time queries. Specifically native regresses on:

- adversarial-pricing-01: 0.875/0.625 vs explicit 1.000/0.875 (-25pp R@5)
- world-knowledge-bridge-01: 0.667/0.437 vs 0.667/0.563 (-13pp R@5)
- negation-and-inference-01: 1.000/0.389 vs 1.000/0.611 (-22pp R@5)

Hypothesis: server-side reasoning persists across `previous_response_id` chains but the *user-visible payload* of each round is just the new probe-result snippets. The agent's internal reasoning has the right concepts but the EXEC-time cue text (which is what we score) is generated from a thinner prompt context. Explicit thinking has the same content sitting in `mt_messages`, so it bleeds vocabulary into subsequent prompts.

Not a clear win for native at gpt-5-mini's reasoning_effort=low. Higher effort or longer chains might shift this.

## EM-ingest substrate experiment

Implementation (subagent `a6a95e82fda3b4b72`): in `mid_execution_eval_e2.py` Phase 1 of `spreading_activation_full`, after each iteration write two events back into the same EM partition:
- `event_type="agent_thinking"`, body=raw assistant output, `speaker="self"`, turn_id starting at 100000.
- `event_type="agent_probes"`, body=joined PROBE lines, turn_id 100001+.

Both branches (multi-turn explicit and native) call `_ingest_agent_round(...)`.

### Unfiltered: catastrophic displacement

Mean coverage drops 0.876→0.717 (−16pp), full_R@5 drops 0.646→0.197 (−45pp). Every single scenario regresses. Inspected `top_hits` for `multi-hop-banquet-01` step 1:

```
rank=1  turn_id=100002  plant_id=null  score=0.7391    (agent_thinking)
rank=2  turn_id=100009  plant_id=null  score=0.7097    (agent_probes)
rank=3  turn_id=2       plant_id=p1    score=0.6747    (REAL PLANT)
rank=4  turn_id=100008  plant_id=null  score=0.6629    (agent_thinking)
rank=5  turn_id=100005  plant_id=null  score=0.6584    (agent_probes)
... (ranks 6-10 also turn_id ≥ 100000)
```

Only rank-3 is a gold plant. The agent's own ingested reflections outscore the chat-form plants by 0.04-0.07 cosine.

**Why displacement happens:** the exec-time cue is generated by the agent at step-write time. Its semantic neighborhood overlaps strongly with the agent's own Phase 1 thinking (same concepts, same internal phrasing, same plan vocabulary). Chat-form plants are framed in conversational dialog ("By the way, the Foundry needs a 120-minute load-in window..."). Cosine similarity inevitably ranks self-similarity above cross-register similarity. The substrate is not just additive — it competes for top-K, and it wins by construction.

**Why coverage also drops:** secondary echo-chamber effect. The agent at exec time reads its own past thoughts back instead of the actual plants, then writes step content reflecting its prior reasoning rather than the underlying facts. Coverage judge marks fewer gold sub-decisions as addressed.

### Filtered: bit-identical to baseline

Added `EM_RETRIEVAL_FILTER_AGENT=1` toggle. At every probe call (both SA-full's `_saf_probe` and the scoring-path probe at line 1822-1830), over-fetch by 3× and drop hits with `event_type ∈ {agent_thinking, agent_probes}`, then truncate to K. Re-ran on the same 10 hard scenarios. Result: **per-scenario numbers identical to baseline at three decimal places**.

Conclusion: when displacement is removed, the substrate provides zero value. The agent never makes productive use of its own ingested reflections, because the only way it would surface them is via embedding-similarity probes — and we just disabled that. The Phase-1 → Phase-3 working-memory continuity hypothesis is empty: the agent's working memory is already covered by `mt_messages` (multi-turn) or `previous_response_id` (native), making EM-ingest redundant for working memory and harmful for episodic retrieval.

### Stratified cognition channel: separate partition, exec-time surfacing

User's architectural prescription: "consider filtering, or different collections/partitions for different modalities — it may not make sense to put messages in the same space as cognition in the same space as pictures in the same space as audio. then the encoding and retrieval of each can be tuned independently."

Implemented via `EM_COGNITION_CHANNEL=1` (`mid_execution_eval_e2.py`):

- New helper `_create_cognition_memory(scenario, ...)`: spins up a *second* `EventMemory` instance per scenario with its own collection (`<base>_cog`) and `SQLAlchemySegmentStore` partition.
- When the channel is on, `_ingest_agent_round` writes Phase 1 THINKING/PROBE events to the cognition partition instead of the main chat partition. Chat retrieval and scoring never see them.
- At Phase 3 per-step exec, *before* the cue-gen probe loop, we probe the cognition partition with the step label as query and surface the top-K=2 hits in a labeled "YOUR PRIOR REASONING" section in `SPREADING_EXEC_STEP_SYSTEM` and `SPREADING_EXEC_WRITE_SYSTEM`. Chat probes (`_saf_probe`) untouched.
- Result: cov **0.838** / full_R@5 **0.569** — better than flat-ingest (no displacement), still net-negative vs baseline (-4pp cov, -8pp R@5).

**Wins (2):** supersession-vendor (+11pp), negative-space (+17pp). Both are scenarios with a META-pattern at exec time — recency-aware retrieval and negative-space inference. The agent's Phase 1 reasoning ABOUT the META-pattern, surfaced at exec time, reinforces the right retrieval/inference frame.

**Losses (7):** stacked-event-planning crashes hardest (-28pp R@5). When Phase 1 anchors on wrong concepts (or partial-correct concepts), surfacing that prior reasoning at exec time keeps the agent on the wrong path even when the chat hits would have rescued it. The cognition channel acts as confirmation bias.

**Architectural conclusion:** stratification is *necessary* (no displacement) but *insufficient*. Unconditional surfacing of prior reasoning at every step is net-negative because Phase 1 reasoning is wrong-anchored on more steps than it is right-anchored on. The next layer needed is **gating** — the cognition channel must not surface unconditionally:

- Agent-decided surfacing (tool-call style: agent asks "what was I thinking about earlier?")
- Confidence-based surfacing (only when chat hits have low max-score)
- Role-specific surfacing (cognition channel might be more useful at PLANNING TIME than at FACT-RETRIEVAL TIME)

The wins-pattern (recency, negation, world-knowledge META) hints that the value is concentrated in *meta-reasoning continuity*, not in fact carry-over. Different memory types serve different *cognitive roles*, and the role determines whether they should be probed at all for a given step.

## Architectural takeaway

The user's framing "memory is cognition's substrate, not passive storage" is right at the *storage* layer (single event log unifies observation, user cognition, agent cognition) but wrong if applied uniformly at the *retrieval scoring* layer. Different memory types play different cognitive roles and demand different retrieval treatments:

| memory type | retrieval role | implementation |
|---|---|---|
| chat history (episodic) | flat-similarity probe | what we have |
| agent thinking (working) | always-in-context, never probed by similarity | `mt_messages` / `previous_response_id` already does this |
| agent thinking (procedural, cross-task) | type-filtered probe surfaces "how I solved a similar task" | not yet tested |
| semantic facts (extracted) | type-filtered probe by entity/relation | adjacent to current EM |

A flat similarity index over a heterogeneous substrate inverts itself: items that share an embedding subspace with the query (the agent's own thoughts) outrank items with the actually-relevant content (the chat history). Type-aware scoring is not optional.

## What might still be worth testing

1. **Cue enrichment via thinking** — concatenate the most recent agent_thinking text into the probe query string, but EXCLUDE agent_thinking events from the hit list. Tests "thinking as input" rather than "thinking as target." Uses both layers but doesn't mix-rank them.
2. **Stratified channels** — probe chat-history with K=5, probe self-history with K=2, surface them in different sections of the agent's exec-time prompt. Tests whether the agent uses self-history when it's clearly labeled.
3. **Procedural memory across tasks** — store completed task transcripts as retrievable for *future* tasks. Tests cross-task reuse, which is where substrate-style memory would be expected to add real value.
4. **Higher reasoning_effort native** — re-test native with `reasoning.effort="medium"` to see if the R@5 regression on adversarial/world-knowledge/negation closes. Cheap variation.

(1) is the most direct test of whether the substrate idea has any value at all in this benchmark shape. (3) is where the architecture is most likely to actually pay off.
